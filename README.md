Dans ce projet, nous avons travaillé sur une base de données fournie par l’entreprise fictive cIAra Mobility, spécialisée dans la location de véhicules électriques partagés en France.

L’objectif était d’exploiter cette base de données à l’aide du langage SQL afin de répondre à différents besoins métier. Nous devions interroger les données, les analyser et construire des requêtes adaptées aux problématiques proposées.

Nous avons utilisé PostgreSQL pour exécuter nos requêtes SQL, ainsi que GitHub pour gérer le versionning du projet.

La base de données est composée de quatre tables principales : les véhicules, les stations, les clients et les locations. Elles sont reliées entre elles grâce à des clés primaires et étrangères, ce qui nous a permis de travailler sur des jointures et de mieux comprendre le fonctionnement d’une base relationnelle.

Le projet est organisé en plusieurs quêtes progressives. Nous avons commencé par des requêtes simples avec SELECT et WHERE, puis nous avons travaillé sur les tris, les jointures entre tables et les fonctions d’agrégation comme COUNT et AVG. Enfin, nous avons réalisé des requêtes plus complètes pour répondre à des cas concrets liés à l’activité de l’entreprise.

Ce projet nous a permis de mieux comprendre la logique du SQL, la structure d’une base de données et l’importance de savoir expliquer et justifier ses requêtes.
